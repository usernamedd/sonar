# Sonar — 录音对话分析工具

> 把对话中的问题，打捞出来。

---

## 一、概述

Sonar 是一个录音对话分析工具。用户录制一段对话（会议、讨论、访谈等），Sonar 将录音转为文字，再用大模型分析对话内容，提取出**一个核心问题**以及**该问题的背景上下文**，最后搜索解决方案并整理输出。

核心价值：**从混沌对话中精准找出一个值得解决的问题，提供可操作的解决思路。**

---

## 二、功能需求

### R1：录音功能

系统支持用户录制音频。

**用户故事**
> 作为用户，我希望录制一段音频对话，以便后续分析。

**验收标准 checklist**
- [ ] 显示录音按钮（开始/停止）
- [ ] 点击开始录音，系统开始录制
- [ ] 点击停止录音，系统保存录音文件
- [ ] 录音文件格式为标准音频格式（WAV/MP3/M4A）
- [ ] 录音文件保存在本地指定目录
- [ ] 保存的录音文件可正常播放

---

### R2：语音转文字（STT）

系统将录音文件转换为文字。

**用户故事**
> 作为用户，我希望把录音转成文字，以便后续分析。

**验收标准 checklist**
- [ ] 输入：音频文件路径
- [ ] 输出：完整文本内容（.txt 文件）
- [ ] 支持中文普通话识别
- [ ] 转写文本可读，无明显遗漏
- [ ] 按说话人或按时长分段，格式清晰

---

### R3：大模型分析——提取问题

系统将转好的文字发送给大模型，分析对话内容，提取出对话中讨论的**一个核心问题**。

**用户故事**
> 作为用户，我希望从对话中提取一个具体问题，而不是泛泛而谈。

**验收标准 checklist**
- [ ] 输入：转写文本
- [ ] 大模型分析对话内容，找出核心问题（仅提取一个）
- [ ] 问题具体明确，不是泛话题
- [ ] 问题可通过后续方案解决
- [ ] 输出：问题描述（一句话）

---

### R4：大模型分析——提取背景

系统同时提取这个问题的背景上下文。

**用户故事**
> 作为用户，我希望知道这个问题是在什么情况下产生的，以便更好地理解问题。

**验收标准 checklist**
- [ ] 输入：转写文本 + 已提取的问题
- [ ] 大模型分析上下文，描述问题产生的场景
- [ ] 输出：背景描述（1-3句话）
- [ ] 背景能让阅读者理解问题从什么情境中产生

---

### R5：搜索解决方案

系统基于提取出的问题和背景，搜索解决方案。

**用户故事**
> 作为用户，我希望有解决方案可参考，而不是只能看到问题。

**验收标准 checklist**
- [ ] 输入：问题 + 背景
- [ ] 搜索来源：互联网搜索（和/或内部知识库）
- [ ] 输出：解决方案列表（每个方案含来源链接）
- [ ] 方案与问题相关，有来源依据
- [ ] 可操作性较强

---

### R6：整理输出

系统将分析结果整理成结构化输出。

**用户故事**
> 作为用户，我希望看到一份结构清晰的报告，包含问题、背景、解决方案。

**验收标准 checklist**
- [ ] 输出包含：问题描述、背景上下文、解决方案列表
- [ ] 输出格式：Markdown 文本
- [ ] 可在 CLI/UI 中展示
- [ ] 输出结构清晰，内容完整，可直接使用

---

## 三、非功能性需求

### 性能
- 录音实时存储，不丢失
- STT 转换时间不超过音频时长的 2 倍
- 大模型分析响应时间不超过 30 秒
- 搜索解决方案不超过 60 秒

### 隐私
- 所有数据本地处理，不上传第三方
- 录音文件和分析结果保存在用户指定目录

### 可扩展性
- STT 模型可替换（默认使用 Whisper 或其他开源模型）
- 大模型可替换（支持 OpenAI / Gemini / 本地模型）
- 搜索来源可扩展（支持更多搜索引擎或知识库）

## 四、技术方案（待细化）

### 整体架构
```
录音 → STT转写 → LLM分析（问题+背景） → 搜索方案 → 整理输出
```

### 技术选型
- 语言：Python（CLI优先）
- STT：Whisper（开源）
- LLM：支持多模型（配置切换）
- 搜索：网络搜索API
- 输出：Markdown
